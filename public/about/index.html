About

Infrastructure as code, Azure, and practical experimentation

I started this site as a place to write down the things I build and the things I learn while building them. Most of my interests sit somewhere around infrastructure as code, Azure, automation, and small software projects that solve a very specific problem without becoming much bigger than they need to be.

I like tools that are clear, dependable, and easy to operate. That applies just as much to cloud infrastructure and CI pipelines as it does to a static website or a script that turns raw data into something useful.

Infrastructure as code Terraform, cloud services, and repeatable setups

I enjoy turning manual infrastructure steps into something versioned, understandable, and easy to run again later.

Application and data experiments Node.js, TypeScript, Python, and lightweight persistence

A lot of my side projects start with a narrow question, then grow just enough to answer it well.

Publishing and documentation Hugo, GitHub Pages, and notes that explain the why

This site is also part of the experiment. I use it to practice writing about projects in a way that is useful to other people and to my future self.
